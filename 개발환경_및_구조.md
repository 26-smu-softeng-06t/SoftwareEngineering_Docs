## 3. 환경변수 운영 기준

실제 `.env`는 개인 로컬 또는 배포 환경의 비밀값 파일이며 Git에 커밋하지 않는다. 공유가 필요한 키 목록과 형식은 `.env.example`만 사용한다.

현재 사용하는 주요 환경변수:

| Key | 용도 | 커밋 가능 여부 |
| --- | --- | --- |
| `APP_ENV` | 실행 환경 | 예시값만 가능 |
| `APP_TIMEZONE` | 서버/앱 기준 시간대 | 예시값만 가능 |
| `MYSQL_HOST` | MySQL 호스트 | 예시값만 가능 |
| `MYSQL_PORT` | MySQL 포트 | 예시값만 가능 |
| `MYSQL_DATABASE` | MySQL DB 이름 | 예시값만 가능 |
| `MYSQL_USERNAME` | MySQL 사용자 | 예시값만 가능 |
| `MYSQL_PASSWORD` | MySQL 비밀번호 | 금지 |
| `MYSQL_SSL_MODE` | MySQL SSL 모드 | 예시값만 가능 |
| `SPRING_DATASOURCE_URL` | Spring JDBC URL | 실제값 금지 |
| `SPRING_DATASOURCE_USERNAME` | Spring datasource 사용자 | 예시값만 가능 |
| `SPRING_DATASOURCE_PASSWORD` | Spring datasource 비밀번호 | 금지 |
| `DATABASE_URL` | 도구용 통합 DB URL | 실제값 금지 |
| `GOOGLE_CLIENT_ID` | Google OAuth Client ID | 실제값 금지 |
| `GOOGLE_CLIENT_SECRET` | Google OAuth Client Secret | 금지 |
| `JWT_SECRET` | JWT 서명 키 | 금지 |
| `ADMIN_SECRET_KEY` | 관리자 승격 비밀키 | 금지 |
| `VITE_API_BASE_URL` | 프론트엔드 API base URL | 로컬 예시값 가능 |

## 4. 설정 파일 연결

### Backend

`backend/src/main/resources/application.yml`은 루트 `.env`를 optional import 대상으로 둔다. 로컬 실행 시 `TOKKI/.env`가 있으면 datasource와 보안 관련 값을 읽을 수 있다.

### Frontend

`frontend/vite.config.js`는 `envDir: '../'`를 사용해 `TOKKI/.env`의 `VITE_*` 변수를 읽는다.

## 5. 보안 주의사항

- `.env`는 Git에 커밋하지 않는다.
- `.env.example`에는 실제 비밀번호, 토큰, OAuth secret, JWT secret, admin secret을 넣지 않는다.
- DigitalOcean/MySQL 접속 문자열을 문서, Issue, PR, Wiki에 그대로 붙여넣지 않는다.
- 비밀값이 노출된 경우 즉시 해당 secret을 폐기하고 재발급한다.
- AI Agent에게 전달하는 프롬프트나 `Onboarding` 파일에는 민감 정보가 섞일 수 있으므로 공개 문서에 원문 복제하지 않는다.

## 6. 로컬 실행 명령 초안

아직 의존성 설치와 CI 명령은 확정 전이다. 현재 구조 기준 예상 명령은 다음과 같다.

```bash
# frontend
cd frontend
npm install
npm run dev

# backend
cd backend
./gradlew bootRun
```

Windows 환경에서는 Gradle wrapper가 추가되기 전까지 로컬 Gradle 설치가 필요할 수 있다.
