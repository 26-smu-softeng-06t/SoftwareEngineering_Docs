## 0. SRS 문서 정의

본 문서는 TOKKI 서비스의 **SRS(Software Requirements Specification)** 기준 문서이다.  
프로젝트의 기능적 요구사항(FR)과 비기능적 요구사항(NFR), 단계별 구현 범위를 단일 기준으로 관리하며, GitHub의 User Story/PBI와 추적 가능하게 연결한다.

- 요구사항 변경은 Issue와 PR로 관리하고, 승인 후 본 문서에 반영한다.
- 초기 일정 계획 및 Sprint 용량 산정은 FR을 기능 단위로 분해한 **FP(Function Point) 스타일 규모 추정**을 참고한다.

## 1. 기능적 요구사항
| ID    | 기능               | 요구사항                                                                                                                         |
| ----- | ------------------ |------------------------------------------------------------------------------------------------------------------------------|
| FR-01 | 회원 관리          | SSO 로그인만 지원. 관리자 계정은 DB role 칼럼으로 구분하며, 관리자 회원가입·로그인 시 Service 단계에서 별도 비밀 키 검증 필요.                                           |
| FR-02 | 난이도 선택        | 하(TOEIC 500↓) / 중(500~800) / 상(800↑) 3단계, 각 난이도별 10단계로 세분화. 퀴즈 시작 전 선택 가능.                                                  |
| FR-03 | 영단어 학습        | 영단어 탭·퀴즈 탭 분리. 단어·뜻·예문을 카드 형식으로 제공. 각 난이도(하/중/상)의 마지막 단계(10단계) 단어에 비즈니스 예문 포함 (Phase 3 구현). + 발음기호 TTS + 단어간 Relation 고려 추천(유의어, 반의어, 동의어, 파생어) + Phrase 및 숙어 포함 |
| FR-04 | 퀴즈               | 한→영 / 영→한 선택 가능, 전 문항 주관식. 중도 이탈 시 문항 단위 자동 저장 후 이어풀기 지원.                                                                    |
| FR-05 | 결과·피드백        | 전체 제출 후 문항별 정오 표시. 오답은 오답노트 자동 기록. 재방문 시 이전 결과 확인 가능. 재학습 후 정답 처리 시 오답노트 해제 여부 사용자 선택.                                       |
| FR-06 | PvP 대결           | 2인 실시간 속도 대결. 방 생성 → 초대 코드 공유 → 시작 흐름. 상대 진행 현황 실시간 표시. 종료 후 점수·시간 비교 결과 제공.                                                 |
| FR-07 | 단어 관리 (관리자) | 단어 개별 등록·수정·삭제. 엑셀(.xlsx) 일괄 업로드 지원 (컬럼: 단어/뜻/예문/난이도, 최대 5MB, 업로드 전 미리보기 제공). 엑셀 파일 첨부 시 DB에 자동 파싱.                          |
| FR-08 | 랭킹               | 전체 사용자 기준 오답 빈도 TOP 10 단어 표시. 매일 자정 갱신.                                                                                      |

## 2. 비기능적 요구사항

| ID     | 분류   | 요구사항                                                                                                |
| ------ | ------ | ------------------------------------------------------------------------------------------------------- |
| NFR-01 | 반응성 | 다음 문항 전환 0.5초 이내. 초기 데이터 로딩 2초 이내. 단, AI 추천(Relation·Phrase) 데이터는 캐싱 미적용 최초 1회에 한해 예외 허용.                    |
| NFR-02 | 보안   | SSO 기반 인증으로 비밀번호 직접 저장 없음. 관리자 비밀 키는 서버 환경변수로 관리. 관리자 기능은 role 권한 검증 후 접근. JWT 토큰 만료 시 자동 로그아웃. |
| NFR-03 | 가용성 | 반응형 웹(모바일 브라우저 지원). 한 손 엄지 조작 가능한 UI. 네트워크 단절 시 진행 상태 로컬 임시 저장.  |
| NFR-04 | 정확성 | 단어·뜻·예문은 공신력 있는 사전(Oxford, Merriam-Webster 등) 기반 검수. 채점 시 대소문자·앞뒤 공백 무시. |

## 3. 개발 단계 (Phase)

| Phase         | 포함 기능                                                                               |
| ------------- | --------------------------------------------------------------------------------------- |
| Phase 1 (MVP) | SSO 로그인, 관리자 role·비밀 키, 단어 탭(카드), 난이도 선택, 주관식 퀴즈, 오답노트, 관리자 단어 CRUD |
| Phase 2       | 엑셀 업로드, 랭킹, 퀴즈 이어풀기, 피드백 고도화, 발음기호 TTS, 단어 Relation 추천, Phrase·숙어 |
| Phase 3       | PvP 대결, 비즈니스 예문, 오프라인 임시 저장                                             |
